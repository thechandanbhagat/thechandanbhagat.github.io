Getting Started

Installation

ezPM2GUI requires Node.js 18+ and PM2 to be installed globally. Install both with:

# Install PM2 globally (skip if already installed)
npm install -g pm2

# Install ezPM2GUI globally
npm install -g ezpm2gui
If you get a permission error on Linux/macOS, prefix with sudo or configure npm to use a user-writable directory.

First Launch

1

Start the GUI server

Run ezpm2gui from any terminal. The server starts on port 3101 by default.

2

Open the dashboard

Navigate to http://localhost:3101 in your browser.

3

Explore your processes

All PM2 processes on the local machine are listed immediately — no configuration needed.

To run ezPM2GUI as a background service on Linux, use the provided ezpm2gui.service systemd unit file.

Dashboard

Overview

The main dashboard gives you an at-a-glance view of all running processes. Each process card shows:

🟢

Status

Online, stopped, errored, or one-launch process state with live color indicator.

📊

CPU & Memory

Real-time CPU percentage and heap memory usage updated every 2 seconds.

♻️

Restarts

Total restart count — useful for detecting crash loops.

⏱️

Uptime

Human-readable uptime since the process last started.

Live Metrics

The Monit page shows real-time performance charts per process. Charts refresh automatically via WebSocket — no manual polling needed.

CPU usage spikes above 80% turn the bar red; 60–80% shows orange. Memory thresholds follow the same pattern.

Process Management

Start / Stop / Restart

Every process card has action buttons:

ActionDescriptionShortcut
StartStart a stopped processClick the play icon on the card
RestartGracefully restart with zero downtime (cluster mode)Click the refresh icon
StopStop a running process (keeps it in PM2 list)Click the stop icon
DeleteRemove process from PM2 list entirelyClick the trash icon (confirmation required)

Viewing Logs

Click any process name to open the Process Detail page, then select the Logs tab. You can:

1

Switch log type

Toggle between stdout and stderr logs using the tab selector.

2

Adjust line count

Use the line count control (50 / 200 / 500 / All) to fetch more history.

3

Download full log

Click the Download button to save the complete log file to your computer.

Log file paths are read directly from PM2's process descriptor. If PM2 was started without log paths configured, logs may be unavailable.

Cluster Mode

Navigate to Cluster Management in the sidebar to scale processes across CPU cores. You can:

  • Scale up or down the number of instances
  • Perform a rolling reload with zero downtime
  • Monitor per-instance CPU/memory breakdown

Remote Servers

Add a Connection

Go to Settings → Remote Connections and click Add Connection. Fill in:

FieldNotes
NameA friendly label shown in the server switcher
HostIP address or hostname of the remote server
PortSSH port (default: 22)
UsernameSSH user (must have PM2 access)
Auth typePassword or private key (PEM format)
Passwords are encrypted at rest using AES-256-GCM before being saved to the config file. Private keys are never stored — only referenced by path.

Switch Servers

Use the server switcher in the top navigation bar (the globe/server icon) to instantly switch context between the local machine and any connected remote server. All dashboard data, logs, and metrics will update to reflect the selected server.

PM2 must be installed on the remote server and accessible in the SSH user's PATH. If detection fails, use the Install PM2 button inside the connection settings.

Deploy Application

The Deploy page allows you to run predefined deployment scripts from the GUI. To set one up:

1

Create a project config

Go to Settings → Projects and add a new project with its name, working directory, and deploy script path.

2

Trigger a deploy

Open the Deploy page, select the project, and click Deploy. Output streams live to the log panel.

3

Review deployment history

Each deployment is logged with timestamp, exit code, and output in the deployment log file.

Cron Jobs

The Cron Jobs page lets you schedule recurring tasks without touching a crontab. Jobs are stored in src/server/config/cron-jobs.json and run inside the ezPM2GUI server process.

🕐

Schedule

Use standard 5-field or 6-field cron expressions (seconds supported).

💻

Command types

Shell commands, Node.js scripts, Python scripts, and .NET executables are all supported.

📋

Run log

Each execution result (exit code, stdout, stderr) is saved for review.

Cron jobs run as the same OS user as the ezPM2GUI server process. Ensure the user has required permissions for the commands being executed.

Settings

Checking for Updates

Go to Settings → Updates. Click Check for Updates — ezPM2GUI queries the npm registry and compares your installed version against the latest published release.

If an update is available, an Install button appears. The installation runs npm install -g ezpm2gui@latest and streams output live. After a successful install, click Restart Server to apply the update.

A pulsing yellow badge also appears in the top navbar whenever a newer version is detected (checked automatically 4 seconds after page load).

Theme

Click the / icon in the top-right of the navigation bar to toggle between dark and light mode. Your preference is applied instantly across all pages.

FAQ

ezPM2GUI says "PM2 not connected" on startup

This usually means PM2 isn't running any processes. Start at least one process with pm2 start <app> and refresh the page.

Remote server shows "PM2 not found"

PM2 is installed in a non-standard PATH on the remote machine. ezPM2GUI tries several detection methods automatically. If all fail, use the Install PM2 button in the connection settings, or SSH in manually and run npm install -g pm2.

Can I run multiple instances of ezPM2GUI?

Yes — set a different port with the PORT environment variable: PORT=3102 ezpm2gui.

How do I run ezPM2GUI as a system service?

On Linux, copy the provided ezpm2gui.service file to /etc/systemd/system/, then run:

sudo systemctl daemon-reload
sudo systemctl enable --now ezpm2gui

On Windows, run install-windows-service.js with Node.js (requires administrator privileges).
